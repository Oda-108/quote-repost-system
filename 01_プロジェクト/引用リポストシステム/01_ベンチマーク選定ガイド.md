# ベンチマーク選定ガイド

引用リポスト自動化システムで監視する10〜15アカウントの選定基準と手順。

---

## 選定基準

| 基準 | 条件 | 理由 |
|------|------|------|
| フォロワー数 | 5,000〜100,000 | 自分の10倍以内がエンゲージメント的に釣り合う |
| ジャンル | ユーザーが自由に設定 | 自分の思想と絡められるジャンルを選ぶ |
| 投稿頻度 | 1日3〜10回 | 引用の機会が十分にある |
| エンゲージメント率 | 3%以上 | 元ポストにインプが集まらないと引用しても意味がない |
| 思想的親和性 | 高〜中 | 引用で自分の色を出せる相手 |
| 除外 | 炎上系・政治系 | リスク回避 |

---

## 選定プロセス

### 手動選定（プロトタイプ）

1. 自分のジャンルに近いアカウントをリストアップ（候補30〜50）
2. 各アカウントのプロフィール・直近10ポストを確認
3. 上記基準でフィルタリング → 15アカウントに絞る
4. アカウントプロファイルテンプレートに記入
5. DynamoDBに登録

### Grok自動選定（将来フェーズ）

Grok APIに以下を入力:

```
ジャンル: {ユーザー設定}
思想軸: {ユーザーのコアメッセージ}
フォロワー範囲: 5,000〜100,000
エンゲージメント率: 3%以上
除外: 炎上系・政治系
```

Grokが候補15アカウントを返却 → ユーザーが確認・承認 → DB登録

## Xでの探し方（具体的な手順）

### 方法1: 検索コマンド

Xの検索窓に以下を入力して関連アカウントを発見:

```
{ジャンルキーワード} min_faves:100 lang:ja
```

例:
```
AI副業 min_faves:100 lang:ja
マーケティング min_faves:100 lang:ja
```

### 方法2: フォロー関係から辿る

1. 既に知っているベンチマーク候補のフォロー先を確認
2. 「この人がフォローしている人」は同ジャンルの確率が高い
3. そこから基準でフィルタリング

### 方法3: For Youフィードの活用

1. 自分のFor Youフィードに流れてくるアカウントを記録
2. Xのアルゴリズムが「似た層」として判定している = 引用との相性が高い

---

## 選定後のアクション

1. 各アカウントのプロファイルを `02_アカウントプロファイルテンプレート.md` のフォーマットで記入
2. DynamoDB `QuoteRepost_AccountProfiles` に登録
3. 月次で見直し
   - エンゲージメント率が落ちたアカウントは入れ替え
   - 新たに伸びているアカウントを追加

---

## 注意事項

- 同じジャンルに偏りすぎない（3〜4のサブジャンルに分散）
- 自分と思想が真逆すぎるアカウントは避ける（引用が攻撃的に見える）
- フォロワー数が少なすぎるアカウントは引用してもIMP伸びない
- アカウントが凍結・休止した場合は速やかに入れ替え
